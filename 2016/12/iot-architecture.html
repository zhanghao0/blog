IoT Platform Architecture

December 4, 2016

History

  • Data Center Management System
  • Smart Streetlighting System

Recently I am responsible for designing an IoT platform for managing various sensors, devices and applications.

Architecture

  1. Multi-agents
  2. Micro-services

Features

  • Distributed
    Unlike centralized NMS, which has one big backend application, the IoT platform uses multiple agents to collect data from various data sources. Load balancer may be utilized to distribute loads of the agents.
  • Self-Manageable
    The IoT platform consists of a Management Platform and an Admin Platform. The former monitors and controls a broad range of devices, sensors and applications, while the later monitors and controls the Management Platform.

Functions

The functions of a typical Network Management System (NMS) have five parts: fault management, configuration management, accounting management, performance management and security management, or FCAPS.

Data Processing Flow

  • Data Source
    An IoT platform deals with data from multiple sources:
    • Things: sensors, devices, applications, services, etc.
    • Users: administrators, operators, tenants, etc.
    • APIs: event, alarm, asset, oAuth, etc.
  • Data Collection & Access
    An IoT platform collects data and allows access to the data using diverse protocols and data formats:
    • Protocols: REST, SNMP, XMPP, MQTT, LoRa, etc.
    • Data Formats: XML, JSON, CSV, Binary, etc.
  • Data Transformation
    Once data are collected, they will be transformed before storing into the database. Many transformation steps may be performed, such as validation, filtering, conversion and enrichment.
